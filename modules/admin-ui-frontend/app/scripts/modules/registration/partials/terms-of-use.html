Opencast is free, open-source software - so why do we bother you with registration and feedback? Well, data on the usage of Opencast is important for us. When it comes to discussions with hardware vendors about cooperation and compatibility they want to know how large the market is before they make any decision. Likewise, public funding for further development relies on such data. Finally, we also need the feedback to further develop Opencast in line with its usage.

Note that you can choose which data you want to provide and do not have to agree to sending data from all categories.

Data Stewardship

Location

This data will be stored within the European Union, and will be GDPR compliant. The current location can be found here

Storage Duration

Organizational Profile

Your organizational profile will be purged 6 months after its last update.

Usage statistics and technical data

Your technical statistics will be purged 24 months after their last update

Access to Information

Acess to the raw data is restricted to the Opencast board, who can authorize release to commiters on a case-by-case basis. Aggregated summaries of the data will be made available from time to time at locations of the board's choosing.

What Happens With Your Data

Personal/Organizational Data

If any data is provided, we will publish the name of your institution on opencast.org and set a geographical marker for your institution on the Opencast adopter’s map.

Usage Statistics and Technical Data

This data itself is anonymized and will be stored without any relation to your organizational or personal data unless you allow us to link this information. The data will be sent in a regular interval. The data sent may include the following information:

  • A timestamp at which this data has been created
  • An optional identifier marking the data source to link your organizational data to your install
  • The number and status of events in your Opencast system
  • The total duration of all media in your Opencast system
  • The number of series in your Opencast system
  • The number of Opencast users (does not collect info from external user providers
  • The number of capture agents connected to your Opencast system
  • The number of jobs in your Opencast system
  • The number of tenants in your Opencast system
  • The version of your Opencast system
  • Technical data regarding servers in your Opencast system
    • A unique anonymous identifier for each server
    • The hostname for each server
    • The number of CPU cores of each server
    • The system memory of each server
    • The disk space of each server
    • The maximum load value for each server
    • The Opencast services running on each server

Error Reports

Additionally, error reports can be automatically sent so that we can monitor if there are systematic errors within Opencast that were not detected in our QA.

Note that this specific data will be linked to your organization so that we are able to warn you in case we found a critical behavior. The error reports may also contain information about your content. This data will not be public. It will only be accessible to Opencast committers.

The data sent may include the following information:

  • Stack traces
  • Error messages
  • A timestamp when the error occurred
  • The Opencast version

Finally, you can change the data you provided at any time. To do so, go to the “Opencast Registration” in the menu “Help” of the administrative user interface. If you deleted your Opencast installation, want to remove your account but cannot anymore, please write an email to support@opencast.org

Thanks!

We highly appreciate any feedback, as even some of the data that we gather about the Opencast usage will help us to create better software, have better chances for additional funding and attract more companies to provide services around Opencast.
